# 接水樂園 遊戲設計文件

## 概覽

單一 HTML5 Canvas 檔案的觸控遊戲。玩家按住螢幕讓水流入杯子，放手時水位若落在目標線附近即過關；水溢出則 Game Over。

---

## 技術規格

- **平台**：HTML5 Canvas，單一 `index.html`，不需任何框架或 build 工具
- **目標裝置**：手機瀏覽器（支援 Touch）、桌面瀏覽器（支援 Mouse）
- **Canvas 邏輯尺寸**：390 × 844（iPhone 比例）
- **縮放**：`devicePixelRatio` HiDPI 處理，所有位置用比例值定義

## 杯子表情

| 水位 | 表情 | 說明 |
|------|------|------|
| 0–60% | 😊 | 開心大笑 |
| 60–85% | 😰 | 緊張皺眉 |
| 85–99% | 😱 | 驚慌張嘴 |
| 達到目標（WIN） | 🌟 | 星星眼慶祝 |
| 溢出（GAME OVER） | 😵 | 暈倒 |

## 水的物理模型

- `fillLevel`：`0.0`（空杯）→ `1.0`（杯緣）
- `targetLevel`：第一關 `0.6`（60% 水位）
- **注水速率**：`+0.002 / frame`（@60fps 約 5 秒從空到目標 60%，約 8 秒從空到滿杯）
- **放手後**：水位立即停止（無慣性）

### 視覺動畫

- **水面波動**：3–4 個正弦波峰；按住時振幅大，放手後衰減
- **水柱**：從滑道末端落下半透明圓形水珠（直徑 8–12px，±3px 隨機偏移），最多同時 30 顆
- **水花**：水珠碰水面產生 2–3 個擴散圓圈，0.3 秒後消失

## 遊戲狀態機

```
READY → PLAYING → WIN
                → GAME_OVER
```

| 狀態 | 觸發條件 | 畫面表現 |
|------|---------|---------|
| `READY` | 初始 | 「按住螢幕開始」提示 |
| `PLAYING` | 首次 pointerdown | 正常遊玩 |
| `WIN` | 放手時 `fillLevel` 在 `targetLevel ± 0.05` 內 | 杯子跳動、星星眼、「完成！」 |
| `GAME_OVER` | `fillLevel >= 1.0` | 水花飛濺、暈倒表情、「再試一次」 |

---

## 勝利判定

- 放手後立即判定
- `targetLevel - 0.05 ≤ fillLevel ≤ targetLevel + 0.05` → WIN
- `fillLevel > targetLevel + 0.05`（超過容許範圍但未溢出）→ 繼續 PLAYING，但目標已不可達，只剩 Game Over 一途
- `fillLevel >= 1.0` → GAME_OVER

---

## 輸入事件

| 事件 | 行為 |
|------|------|
| `touchstart` / `mousedown` | 開始注水，進入 PLAYING |
| `touchend` / `mouseup` / `mouseleave` | 停止注水，立即判定勝負 |

## 程式碼架構

```
index.html
 └── <script>
      ├── 常數設定（FILL_RATE, TARGET_LEVEL, 顏色）
      ├── state 物件（fillLevel, gameState, isPouring, particles[]）
      ├── 繪圖函式
      │    ├── drawBackground()    背景 + 機身 + 滑道
      │    ├── drawCup()           杯身 + 目標線
      │    ├── drawWater()         水位 + 波動
      │    ├── drawFace()          表情
      │    ├── drawParticles()     水珠 + 水花
      │    └── drawUI()            狀態列 + 提示文字
      ├── 更新函式
      │    ├── updateWater()
      │    └── updateParticles()
      ├── onPointerDown() / onPointerUp()
      ├── checkWinLose()
      └── gameLoop()              requestAnimationFrame
```
