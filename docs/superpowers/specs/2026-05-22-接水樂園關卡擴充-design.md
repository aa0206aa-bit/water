## 2. 杯型描述

每個 `cup` 物件包含：

```js
{
  type: string,
  widthAt(t),   // t=0 底部, t=1 頂部，回傳該高度的杯內寬度（px）
}
```

各杯型的 `widthAt(t)` 函數：

| 杯型     | 函數                              |
|----------|-----------------------------------|
| 圓杯     | 100（固定，現有形狀）             |
| 直筒杯   | 90（固定）                        |
| 寬口杯   | 70 + 50 * t                       |
| 錐形杯   | 120 − 50 * t                      |
| 沙漏杯   | 60 + 55 * Math.pow(Math.abs(t - 0.5) * 2, 2) |
| 不規則杯 | 85 + 20 * Math.sin(t * Math.PI * 3) |

## 3. 畫面流程

新增 `'SELECT'` gameState，完整狀態機：

```
SELECT ──點選已解鎖關卡──▶ READY ──按住──▶ PLAYING
                                              │
                          ┌───────────────────┤ 放開
                          ▼                   ▼
                        WIN               GAME_OVER
                          │                   │
                     下一關/回選關        再試/回選關
                          └────────▶ SELECT ◀─┘
```

- 啟動時讀 localStorage，直接進入 `SELECT`
- `WIN`：顯示「下一關 →」＋「選關」；末關只顯示「再玩」
- `GAME_OVER`：顯示「再試一次」＋「選關」

---

## 4. 關卡選擇畫面

- 2 × 3 格子，每格約 140 × 160px，置中於畫布
- 每格內容：縮小版杯型（canvas 繪製）＋ 關卡編號
- 狀態視覺：
  - 已過關：深色邊框 + 星號標記
  - 已解鎖未過：正常邊框，可點
  - 鎖定：半透明灰色，不可點

---

## 5. 繪圖改動

### cupOutline(cup, x, y, w, h)
取代現有 `cupPath()`。對左右各採樣 N 點，按 `cup.widthAt(t)` 動態畫杯壁。

### drawWater(cup, x, y, w, h, fillLevel)
從底部到 `fillLevel` 高度，每行寬度依 `cup.widthAt(t)` 決定。頂部保留波浪動畫。

### drawTargetLine(cup, x, y, w, h, target)
在 `t = target` 高度，按當時杯寬畫虛線。

### drawCupMini(cup, cx, cy, w, h)
關卡選擇畫面用的縮小版杯型，複用 `cupOutline` 邏輯。

## 6. 進度儲存

- Key：`'waterGame_maxUnlocked'`（localStorage）
- 值：0–5 的 index，預設 0
- 通關後：`maxUnlocked = Math.max(maxUnlocked, currentLevel + 1)`
- 無分數、星級等額外儲存
