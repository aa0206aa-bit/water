# 集水站設計文件

## 概述

在滑道出口與水杯之間加入「集水站」中間容器。玩家按住時，水粒子落入集水站累積；放開後，集水站自動將水倒入水杯，倒完後判定勝負。

---

## 版面常數

| 常數 | 舊值 | 新值 | 說明 |
|------|------|------|------|
| `BLOCKER.y` | 515 | 475 | 往上移，騰出集水站空間 |
| `SPIN_ARM` | 55 | 35 | 旋轉臂縮短，避免與集水站重疊 |
| `CUP.y` | 595 | 615 | 往下移，留出倒水流視覺距離 |
| `COLLECTOR` | — | `{ x:155, y:515, w:80, h:45 }` | 新增；中心 x=195 對齊水杯中心 |
| `POUR_RATE` | — | `1/90 ≈ 0.0111` per frame | 1.5 秒倒完滿桶（60fps 基準） |

垂直層次：
```
y=435  滑道出口（粒子產生點）
y=475  擋板中心（旋轉臂覆蓋 y=440–510）
y=515  集水站頂端
y=560  集水站底端
y=570  出水嘴底端
y=615  水杯頂端
y=735  水杯底端
```

## 狀態機

### 新增狀態：`POURING`

```
READY
  ↓ 按下螢幕
PLAYING   粒子落入集水站
            collectedLevel >= 1 → GAME_OVER（裝太多）
  ↓ 放開螢幕
POURING   每幀自動轉移水量
            fillLevel >= 1 → GAME_OVER（水杯溢出）
            collectedLevel <= 0 → checkWinLose()
  ↓
WIN / GAME_OVER
```

### 新增狀態變數

- `state.collectedLevel`（0–1）：集水站當前蓄水量
- `resetLevel()` 與 `resetGame()` 均重置為 0

### 各函式修改

| 函式 | 修改內容 |
|------|---------|
| `updateWater` | 新增 POURING 分支：每幀 `collectedLevel -= POUR_RATE`、`fillLevel += POUR_RATE`；溢出或倒完時切換狀態 |
| `updateParticles` | 碰撞目標改為 COLLECTOR；命中時 `collectedLevel += rate/2`；移除舊的 CUP 碰撞邏輯 |
| `onPointerUp` | 放開後設 `gameState = 'POURING'`（原本直接呼叫 `checkWinLose`） |
| `checkWinLose` | 狀態守衛改為 `gameState !== 'POURING'` |

---

## 遊戲規則

- **PLAYING 中集水站溢出**（collectedLevel >= 1）→ 立即 GAME_OVER，提示「水太多了！」
- **POURING 中水杯溢出**（fillLevel >= 1）→ GAME_OVER，提示「水滿了！」
- **倒完後判定**：`|fillLevel - target| <= tolerance` → WIN；否則依 fillLevel 顯示「水太少了！」或「水滿了！」
- 目標線**只顯示在水杯**，不顯示在集水站（玩家需自行估算換算量）

## 視覺設計

### `drawCollector()`
- 圓角矩形（r=6），米色底 `rgba(230,222,210,0.9)`，金色邊框
- 底部中央小出水嘴（梯形，寬 14→8px，高 10px）
- 上方 9px 字體標籤「集水站」

### `drawCollectorWater()`
- 藍色漸層（`#7ec8e3` → `#5ab0d0`），clip 在集水站內部
- 靜態水面（無波浪，容器小不明顯）
- `collectedLevel <= 0` 時跳過

### `drawPourStream()`
- 僅在 `POURING` 且 `collectedLevel > 0` 時繪製
- 從出水嘴底端（y≈570）到水杯當前水面（`CUP.y + CUP.h * (1 - fillLevel)`）
- 藍色漸層矩形，寬度 = `max(2, 6 * collectedLevel)`（快倒完時流變細）

### 繪製順序（loop 內）
```
drawCupFill → drawWater → drawPourStream
→ drawCollector → drawCollectorWater
→ drawBlocker → drawParticles
→ drawCupBorder → drawTargetLine → drawFace
```

### READY 提示文字
`按住螢幕開始集水` （原本：`按住螢幕開始注水`）

## 粒子系統調整

`updateParticles` 中集水站碰撞條件：

```javascript
if (state.gameState === 'PLAYING' &&
    p.y >= COLLECTOR.y &&
    p.x >= COLLECTOR.x && p.x <= COLLECTOR.x + COLLECTOR.w) {
  spawnSplash(p.x, COLLECTOR.y);
  state.collectedLevel = Math.min(1, state.collectedLevel + lvl().rate / 2);
  if (state.collectedLevel >= 1) {
    state.gameState  = 'GAME_OVER';
    state.loseReason = 'tooMuch';
    state.isPouring  = false;
  }
  return false;
}
```

移除原有 CUP 碰撞邏輯（`waterTop` / `cx`/`cy` 等 CUP 解構變數一併清除）。

## drawUI GAME_OVER 文字補充

新增 `tooMuch`（集水站溢出）的顯示分支：

| loseReason | title | hint |
|------------|-------|------|
| `tooLittle` | 水太少了！ | 多撐一會兒 |
| `tooMuch` | 集太多了！ | 少集一點 |
| `overflow` | 水滿了！ | 放手快一點 |

實作方式：將原本的三元運算子改為三路條件。
