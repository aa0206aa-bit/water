# Water Game — 設計文件

**日期：** 2026-05-18  
**平台：** 瀏覽器（HTML5 Canvas）  
**技術棧：** 純 HTML5 Canvas + Matter.js（剛體）+ 自製粒子水流  
**MVP 範圍：** 3 關

---

## 1. 遊戲概述

2D 物理導流益智遊戲。玩家在畫面上用滑鼠/手指繪製線條，線條立即成為具物理特性的實體，用來引導從水源流出的水粒子，導入目標容器達到填充線即過關。

---

## 2. 核心機制

### 2.1 畫線系統
- 滑鼠按下 → 開始記錄路徑點（每 5px 取樣一次）
- 滑鼠放開 → 路徑轉成 Matter.js 複合靜態剛體（由多段細矩形拼接）
- 最短有效線段：20px（防止誤觸）
- 點擊已畫線條可刪除，墨水量歸還

### 2.2 墨水限制（Ink Limit）
- 頂部顯示墨水條（條狀漸層）
- 畫線越長消耗越多墨水
- 墨水耗盡後無法繼續畫線
- 墨水使用量與星級掛鉤（見 2.5）

### 2.3 水源觸發
- 玩家點擊水源按鈕 → **僅能觸發一次**
- 一次噴出固定數量水粒子（60 顆）
- 水噴完後不能再按
- 若水量不足以過關，只能重置整關重來
- 設計意圖：迫使玩家「先規劃、再觸發」

### 2.4 水粒子物理
- 每 80ms 從水源噴出一顆粒子
- 粒子半徑：6px
- 自製重力加速度、速度更新、與 Matter.js 剛體的圓形碰撞偵測
- 粒子上限：150 顆（效能保護）
- 落出畫面底部 → 自動回收

### 2.5 勝負條件與星級
- **過關：** 容器內水粒子達到虛線填充標記
- **失敗：** 水噴完後水量不足（只能重置）
- **星級評等：**

| 星級 | 條件 |
|------|------|
| ⭐⭐⭐ | 消耗墨水 ≤ 33% |
| ⭐⭐ | 消耗墨水 ≤ 66% |
| ⭐ | 消耗墨水 ≤ 100% |

## 3. 視覺設計（日系奶油風）

### 3.1 色調
| 元素 | 色碼 |
|------|------|
| 背景 | `#FFF8F0`（米白）+ 淡格線 |
| 水粒子 | `#89CFF0`（柔藍），邊緣透明，堆疊時疊色 |
| 玩家畫線 | `#C8A882`（溫暖米棕） |
| 管道/牆壁 | `#B89A7A`（奶茶色） |
| UI | 圓角、柔和陰影 |

### 3.2 杯子表情系統
| 水量狀態 | 表情 |
|----------|------|
| 空 | 擔心 😟 |
| 50% | 期待 👀 |
| 80% | 開心 😊 |
| 100%（過關） | 大笑 😄 |

### 3.3 水粒子視覺
- 半徑 6px 圓形，顏色 `#89CFF0`
- 粒子間距近時疊色（模擬液體聚合感）
- 透明度疊加取代 metaball（效能考量）

## 4. 架構

```
~/game/water/
├── index.html
├── style.css
├── js/
│   ├── main.js          # 遊戲主循環、場景切換
│   ├── physics.js       # Matter.js 封裝（剛體管理）
│   ├── water.js         # 自製粒子水流系統
│   ├── drawing.js       # 畫線輸入 → 剛體轉換
│   ├── ui.js            # 墨水條、星星評等、按鈕
│   └── levels/
│       ├── level1.js
│       ├── level2.js
│       └── level3.js
├── assets/
│   └── （杯子表情圖、音效等）
└── lib/
    └── matter.min.js
```

### 資料流
1. 玩家畫線 → `drawing.js` 捕捉路徑 → 轉成 Matter.js 靜態剛體 → 扣除墨水
2. 玩家觸發水源（一次）→ `water.js` 噴出 60 顆粒子 → 自製重力 + 碰撞 → 匯集至容器
3. 容器水量達虛線標記 → 計算用墨量 → 顯示 1–3 星

## 5. 關卡設計

| 關卡 | 設計重點 | 主要挑戰 |
|------|----------|----------|
| Level 1 | 水源在上、杯在正下方，一條斜線即可過 | 教學關，熟悉畫線與水源觸發 |
| Level 2 | 中間有障礙物，需繞路 | 引入擋板概念 |
| Level 3 | 水源偏左、杯在右下，管道有缺口 | 需多條線配合 |

---

## 6. UI 元素

- **頂部：** 墨水條（漸層顯示剩餘量）、返回鍵、重置鍵
- **水源：** 可點擊按鈕，觸發後變為灰色（不可再按）
- **過關畫面：** 星星動畫 + 下一關按鈕

---

## 7. 技術決策記錄

| 決策 | 選擇 | 理由 |
|------|------|------|
| 遊戲框架 | 無（純 HTML5 Canvas） | 輕量、無依賴 |
| 剛體物理 | Matter.js | 成熟穩定，省去碰撞系統開發時間 |
| 水流模擬 | 自製粒子系統 | 保持視覺彈性，可調出奶油可愛感 |
| Metaball | 不實作 | 效能成本過高，透明度疊加足夠 |
| 水源機制 | 單次觸發 | 提升策略性，迫使玩家先規劃 |
